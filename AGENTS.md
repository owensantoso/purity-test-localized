# Project Instructions

This is a small project. Keep changes surgical and documentation lean.

## Product Rules

- Do not change the canonical item order, IDs, or scoring behavior unless the user explicitly asks.
- Treat translations as faithful translations. Do not localize by replacing concepts, softening tone, or modernizing the test.
- Keep source-language content separate from translated content.
- Do not collect result data by default. Any stats feature must be opt-in and aggregate-only.

## Engineering Rules

- Prefer data files over hard-coded translated strings.
- Use item IDs, not text, for scoring and saved answers.
- Add validation when adding or changing language content.
- Avoid speculative features until the MVP is working.
